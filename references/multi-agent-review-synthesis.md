# 多Agent评审合成

当多个 Agent 各自独立评审同一文档，如何合成最终意见？

## 合成策略

1. **优先级投票**：高优先级问题（P0/P1）自动覆盖低优先级
2. **去重合并**：相似问题合并为一条，标注来源
3. **冲突仲裁**：两个 Agent 意见矛盾时，人工仲裁或置信度高的胜出
4. **行动清单**：输出格式化的 `[x] 文件 | P0/P1/P2 | 具体修改`

## 输出模板

```
## 评审合成报告

### P0 问题（必须修）
- [ ] `文件.md` Lxx | 描述 | 建议

### P1 问题（建议修）
- [ ] `文件.md` Lxx | 描述 | 建议

### 建议
- Agent A：...
- Agent B：...
- 合成意见：...
```

## 注意事项

- 每个 Agent 的评审意见需**独立保存**（不改原始意见）
- 合成时需标注**每个问题的来源 Agent**
- 最终决定权在用户（G先生）
